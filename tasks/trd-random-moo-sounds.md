# TRD: Random Moo Sounds on Bull Tap

Companion to [prd-random-moo-sounds.md](prd-random-moo-sounds.md). Covers the concrete implementation: files touched, the audio module rewrite, data flow, edge cases, and the test plan.

## 1. Overview

`playMoo()` in `src/audio/useFarmAudio.ts` today owns a single `Howl` over `moo.wav`. We replace that one Howl with a **pool of three Howls** (one per real MP3), add a **no-immediate-repeat picker**, **per-clip gain**, and **cut-off-before-play**. The only call site (`BullHerd.tsx`) is unchanged — `playMoo()` keeps its signature. Mama Tobu never calls `playMoo()`, so no mascot change is needed.

```
BullHerd onClick(instanceId) ──> playMoo()
                                    │  (mute gate + iOS unlock, unchanged)
                                    ├─ stop all three moos   (cut-off, FR-3)
                                    ├─ pick index ≠ lastIndex (no-repeat, FR-1)
                                    └─ moos[index].play()     (per-clip volume, FR-2)
Mascot onClick ──> opens info bubble (no audio) — untouched
```

## 3. `src/audio/useFarmAudio.ts` changes

Replace the single `moo` Howl with an array + selection state:

```ts
// Per-clip gain balances perceived loudness (FR-2). Starting values —
// tuned by ear during dev-browser verification:
//   moo-3 (distant/quiet) boosted; moo-1/2 (close-up) eased.
const MOO_SOURCES: Array<{ src: string; volume: number }> = [
  { src: '/audio/moo-1.mp3', volume: 0.45 }, // universfield, close-up
  { src: '/audio/moo-2.mp3', volume: 0.40 }, // koiroylers, close-up (loudest → most cut)
  { src: '/audio/moo-3.mp3', volume: 0.75 }, // ElevenLabs distant (quietest → boosted)
];

let moos: Howl[] | null = null;
let lastMooIndex = -1;
```

- `ensureSounds()` builds `moos` = `MOO_SOURCES.map(s => new Howl({ src: [s.src], volume: s.volume }))` instead of the single moo.
- `playMoo()` becomes:

```ts
export function playMoo() {
  unlockAudio();
  if (useFarmStore.getState().isMuted) return;
  ensureSounds();
  if (!moos || moos.length === 0) return;

  // Cut-off (FR-3): stop any moo still playing before the new one.
  for (const m of moos) m.stop();

  // No-immediate-repeat pick (FR-1).
  let index = Math.floor(Math.random() * moos.length);
  if (moos.length > 1 && index === lastMooIndex) {
    index = (index + 1) % moos.length;
  }
  lastMooIndex = index;

  moos[index].play();
}
```

Notes:
- The `(index + 1) % n` nudge on collision is uniform enough for n=3 and avoids a reroll loop; acceptable per PRD ("deterministic seeding not required").
- `unlockAudio()` / `syncAmbientPlayback()` / `setAmbientMuted()` are untouched — they only concern `ambient`. The ambient Howl and its logic stay exactly as-is (ambient is out of scope).
- Disposal: Howler holds these three small Howls for the session lifetime, same as the old single moo — no unload needed.

## 6. Edge cases

- **Muted:** `playMoo()` returns before any `stop()`/`play()` — no state change, no residual moo. ✅
- **Pre-unlock (first ever tap):** `unlockAudio()` runs first; Howls are created; the tap both unlocks and plays. ✅
- **Rapid double-tap same bull / two bulls:** second call stops the first clip mid-play, then starts the new one → single voice. ✅
- **n=3 no-repeat:** with three clips there's always a valid non-repeat pick; the `%` nudge guarantees `index !== lastMooIndex`. ✅
- **Mascot tap:** doesn't route through `playMoo()` at all — silence preserved structurally, not by a name check. ✅
- **Load failure of one MP3:** that Howl simply won't produce sound; others still work; no crash (Howler swallows load errors). Low risk given local bundled assets.

## 7. Test plan (dev-browser)

1. **Wiring & no-repeat:** instrument `lastMooIndex` (or log the chosen `src`) via the module graph; tap bulls ~10× and assert (a) a real `moo-N.mp3` plays each time, (b) never the same index twice consecutively.
2. **Mascot silent:** tap Mama Tobu; confirm `playMoo` is not invoked and no moo Howl enters `playing()` state.
3. **Mute gate:** with mute on, tap a bull; assert no Howl is `playing()`. Unmute, tap, assert playback.
4. **Cut-off:** trigger `playMoo()` twice in quick succession; assert at most one moo Howl reports `playing()` after the second call.
5. **Per-clip gain:** read each Howl's `volume()`; confirm the three configured values are applied. (Perceived-loudness balance is a manual listen — noted as such.)
6. **Offline/precache:** production build; confirm the three MP3s appear in the generated `sw.js` precache manifest and `moo.wav` does not.
7. **Regression:** ambient mute/unmute still works (unchanged path); no new console errors; tsc + eslint + build clean.

## 8. Rollback

Pure additive/behavioral swap on one module + assets. Rollback = restore the single-moo `useFarmAudio.ts`, re-add `moo.wav`, revert the vite glob. No data, schema, or Firestore involvement.
