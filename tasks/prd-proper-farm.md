# PRD: A Proper Farm — Farmstead Props + Ground Variety

## Introduction

The pasture today is a fenced lawn: uniform procedural grass, a barn, a signpost, a tractor, and the herd. The request ("make the farm even bigger!") clarified into something better than acreage: **make it read as a proper working farm**. Same size, same herd density, same auto-scaling — but with a farmstead that looks lived-in and ground that has *places* instead of just space.

Two locked directions (from clarification):
1. **Farmstead props** — a silo, hay bales, a water trough, and a well, clustered around the existing barn, in the same procedural low-poly toy style as every other model.
2. **Ground variety** — a dirt path leading to the barn, a small pond, and a few mud/flower patches that break up the uniform grass.

Explicitly rejected in clarification: any change to pasture dimensions or the auto-scaling curve, a fence gate, ambient critters, extra scenery counts, or a second tractor.

## Goals

- The farm reads as a working farmstead at first glance: barn + silo + hay + trough + well composed as one cluster
- The ground has at least three visually distinct features (path, pond, patches) instead of uniform grass
- All new props match the existing art style (flat-shaded, low-poly, procedural GLBs from the same generator pipeline) and palette
- Bulls, the tractor, and spawn placement never clip through or stand inside any new prop or the pond
- Zero change to pasture size, wander bounds, camera limits, or the herd-growth curve
- No measurable frame-rate regression with the full herd

## User Stories

### US-001: Procedural farmstead prop models
**Description:** As a viewer, I want a silo, hay bales, a water trough, and a well near the barn, so the farm looks like someone actually runs it.

**Acceptance Criteria:**
- [ ] Four new procedural models generated by `scripts/generate-models.mjs` using the existing `MeshBuilder` primitives (`box`/`prism`/`cylinder`): **silo** (tall cylinder + roof cap), **hay bale** (rounded block or cylinder, warm straw tone), **water trough** (open-top box shell with a water-blue inner face), **well** (low cylinder + two posts + small roof)
- [ ] Each exports as its own GLB (e.g. `silo.glb`, `hay.glb`, `trough.glb`, `well.glb`) following the `fence.glb`/`wheel.glb` pattern, with `useXModel()` hooks + `useGLTF.preload` in `src/scene/models.ts`
- [ ] Flat-shaded, no textures/normals, palette drawn from the existing material set (wood browns, creams, straw/golden, water blue) — consistent with the barn/tractor at default camera distance
- [ ] Regenerating via `node scripts/generate-models.mjs` produces all GLBs with no manual post-processing (already part of `npm run build`)
- [ ] New GLBs are covered by the existing `models/*.glb` precache glob (verify in the build's precache manifest)
- [ ] Typecheck/lint passes

### US-002: Farmstead cluster placed and protected
**Description:** As a viewer, I want the props composed around the barn like a real farmyard, and as a bull, I should never walk through a silo.

**Acceptance Criteria:**
- [ ] The silo stands beside the barn; hay bales (2–3, varied rotation/scale) near the barn wall; the trough at the barnyard's edge where bulls plausibly drink; the well set slightly apart — one legible farmyard composition around the existing barn at (-8, -6)
- [ ] Each solid prop (silo, well, trough; hay optional if large enough) gets a `LANDMARK_EXCLUSIONS` entry in `BullHerd.tsx` so wandering bulls path around it and the reactive push-out keeps them clear
- [ ] Spawn placement (`computeSpawnPositions`' `isBlocked`) automatically respects the new exclusions (it already uses `insideExclusion`) — verified by reload: no bull spawns inside a prop
- [ ] No prop sits on the tractor's patrol rectangle (waypoints at `bound - 2.5`) at any herd size from 13 to the 42 cap — verify the cluster's footprint stays inside the patrol loop's interior at minimum bound
- [ ] Props are non-interactive scenery: no tap handlers, no raycast fattening (real meshes only, no invisible hitboxes — see the mascot-eclipse hotfix precedent)
- [ ] Camera tracking, bull taps, and landmark taps all still work with the cluster present
- [ ] Typecheck/lint passes
- [ ] Verify in browser using dev-browser skill

### US-003: Ground variety — path, pond, patches
**Description:** As a viewer, I want the ground itself to have features — a path to the barn, a pond, worn patches — so the pasture feels like terrain instead of a green carpet.

**Acceptance Criteria:**
- [ ] A **dirt path** runs from the pasture's edge (fence line side nearest the barn) to the barn's door area — flat, slightly irregular edges, earthy brown, clearly readable as a path from the default camera
- [ ] A **small pond** (roughly 3–5 units across) sits in an open area away from the barn cluster, mascot, signpost, and the tractor's patrol line — flat water-blue surface with a subtle darker rim
- [ ] 3–5 **mud/flower patches** scatter deterministically (seeded, stable across reloads) inside the pasture — small, low-contrast, visibly breaking up the grass without becoming noise
- [ ] The pond gets a `LANDMARK_EXCLUSIONS` entry so bulls never stand in the water (path and patches are walkable — no exclusion)
- [ ] Ground features sit flush (decal-style: tiny y-offset, no z-fighting, no depth-write artifacts — same technique as the tracking ring) or ride the grass shader; either way no flicker at any camera angle/zoom
- [ ] Features stay within the *minimum* pasture bound (±14) or scale/position with `computePastureBound` so they never end up outside the fence at small herd sizes — decide once, deterministically
- [ ] Typecheck/lint passes
- [ ] Verify in browser using dev-browser skill

### US-004: No-regression guard
**Description:** As a user, I want the farm to behave exactly as before — just look better.

**Acceptance Criteria:**
- [ ] Pasture bound, fence ring, camera max distance, pan limit, and the herd-growth curve are byte-identical in behavior (no edits to `computePastureBound` or its constants)
- [ ] Bull tap → bubble, Wall of Fame → camera tracking + ring, pager, tractor patrol + stop-for-bull all verified working with the new props present
- [ ] No new console errors; frame pacing at the current herd size feels unchanged in the preview (no visible stutter introduced by added draw calls)
- [ ] Production build succeeds; precache manifest includes the new GLBs and stays within sane size (each new GLB expected ≤ ~10KB like existing models)
- [ ] Typecheck/lint passes
- [ ] Verify in browser using dev-browser skill

## Functional Requirements

- FR-1: Add `silo()`, `hayBale()`, `trough()`, `well()` builders to `scripts/generate-models.mjs`, exporting `silo.glb`, `hay.glb`, `trough.glb`, `well.glb`.
- FR-2: Add corresponding `useSiloModel()` etc. hooks + preloads in `src/scene/models.ts`.
- FR-3: Render the farmstead cluster in `Farm.tsx` (a `Farmstead` component of positioned `<Clone>`s, following the `Scenery`/`Tractor` composition patterns), positioned relative to the barn.
- FR-4: Add exclusion circles for silo, well, trough, and pond to `LANDMARK_EXCLUSIONS` in `BullHerd.tsx` (radius = footprint + small margin).
- FR-5: Render ground features (path, pond, patches) as flat decal meshes with deterministic seeded placement, or as an extension of the existing grass `onBeforeCompile` shader — implementation's choice, but deterministic and flicker-free.
- FR-6: Keep all pasture-geometry functions in `farmLayout.ts` unchanged.
- FR-7: All new assets flow through the existing generate → GLB → preload → precache pipeline; no runtime-generated geometry beyond simple decal planes.

## Non-Goals

- **No pasture size change** — bounds, growth curve, camera limits, and pan limits stay exactly as they are.
- **No fence gate** and **no ambient critters** (explicitly declined in clarification).
- **No second tractor** and **no scenery-ring count changes** (declined earlier).
- **No interactivity on new props** — no tap handlers, no bubbles, no invisible hitboxes (the mascot-eclipse lesson: oversized hitboxes near bull paths swallow taps).
- **No Firestore/data changes** — purely visual/scene work.
- **No water animation/reflection** — the pond is flat stylized color, matching the toy aesthetic.

## Design Considerations

- **Composition over scatter**: the props should form one believable farmyard (silo tight to the barn, hay near the wall, trough at the yard edge, well slightly apart) rather than evenly-spaced decorations. The barn corner of the pasture becomes the "worked" zone; the open pasture stays the herd's.
- **Palette discipline**: straw/golden tones for hay, weathered wood for the well/trough, muted metal-grey or cream for the silo — nothing that competes with the bulls' full-spectrum coats (the winners are the color story of this farm).
- **Path routing**: the path should read as *used* — from the fence side nearest the barn to the barn door, not geometric center-to-center. Slight bends beat straight lines.
- **Patch restraint**: mud/flower patches are seasoning, not features — low contrast, small, and few (3–5). If they're noticeable from every screenshot, there are too many.

## Technical Considerations

- **MeshBuilder already has `cylinder()`** (added for the tractor wheels) — the silo, well, and round hay bales need no new primitives.
- **Exclusion cost is negligible**: `LANDMARK_EXCLUSIONS` is iterated per-bull per-frame; going from 3 to ~7 entries is trivial at herd scale.
- **Pond placement vs. tractor**: the patrol rectangle sits at `bound - 2.5`, which *moves* with herd growth (±11.5 at min bound, ±39.5 at cap). The pond must be placed where no patrol rectangle for any bound in [14, 42] crosses it — practically, keep it well inside ±9 or accept checking against the current bound only (flagged as an open question).
- **Decals vs. shader for ground features**: decal planes (like the tracking ring: tiny y-offset, `depthWrite: false`) are simpler and independently positionable; extending the grass shader keeps draw calls flat but couples path/pond shapes to shader math. Either satisfies US-003; decals are the lower-risk default.
- **Draw calls**: ~4 GLB clones + ~7 decal planes ≈ a dozen added draw calls — noise compared to the existing scene (fence ring alone is ~32 clones).

## Success Metrics

- A first-time visitor's screenshot of the barn corner shows a recognizable farmyard (silo + hay + trough + well) rather than a lone barn
- At least three distinct ground features visible when panning the pasture
- Zero bull/tractor clipping through any new prop or the pond across a 60-second observation
- tsc / eslint / production build clean; no console errors; no perceptible frame-rate drop

## Open Questions

- Pond position vs. the growing tractor patrol rectangle: pin the pond safely inside the minimum patrol loop (simplest, recommended), or compute its position from the current bound? Defaulting to pinned-safe.
- Hay bales: round (cylinder on its side) or square (boxes)? Round reads more "farm", square is cheaper — implementation-time art call.
- Should the dirt path's fence-end align with where a gate *would* be, to leave room for the declined-gate idea later? Cheap to do; assumed yes.
